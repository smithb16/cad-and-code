Features This system does not fully automate the trip booking process. It instead replaces administrative copy-pasting, generating documents from trip information stored in a central location. The system performs the following:

  • Generates a guide-facing Google Calendar event containing relevant links, client information, and trip details.
  • Generates a Google Form and associated results spreadsheet to collect client information (medical details, climbing experience, etc.)
  • Sends custom trip confirmation email to clients containing guide details, meeting location details, etc.
  • Sends pre-trip reminder emails to the guide, and generates messages for easier guide-client communication.
  • Allows for additional groups and additional guides to be added to trip after initial trip booking.
  • System was developed with minimal coding to facilitate maintenance by guide service administrators rather than software engineers.

Airtable Our Airtable relational databases contain information about our clients, guides, meeting locations, and trip offerings. The Airtable button features allow us to instantly take action on a trip, such as adding a new group to an existing trip. Airtable easily integrates with Zapier.

Zapier Zapier is the glue that connects all pieces in this system. Zapier is a visual API, allowing Airtable, Google Forms, Google Drive, and Gmail to communicate with each other.

Google Apps Script This system uses Google Apps Script, a derivative of JavaScript, for advanced automated editing of Google objects. The script receives JSON objects via POST requests and modifies existing Google Forms and Calendar events accordingly.

Phase 3: Profit This system reduced the per-trip administrative time by 60% and, more importantly, eliminated copy-paste errors. Administrators can manually modify all documents before delivery to the client, useful for one-off needs. The guide service's administrators now use their time for projects that are more interesting than copy-pasting.
